# Paper Survey — 系统性文献综述

> **方法论说明**
> 本Skill用于对某个研究领域进行系统性的论文调研，产出结构清晰、覆盖全面的文献综述报告。核心原则：广泛检索、系统分类、深度对比、趋势提炼。

你正在执行一次系统性文献综述。最终产出一份**排版精美的PDF综述报告**。

## 前置准备

### 环境准备

1. **确认PDF转换脚本可用**：本Skill自带 `scripts/md_to_pdf.py`（基于WeasyPrint），用于将最终Markdown报告转为排版精美的PDF。确保依赖已安装：`pip install weasyprint markdown --break-system-packages`。

### 明确调研范围

拿到用户输入后，确认以下信息。如果用户已经给得足够明确（比如"帮我survey一下multi-agent collaboration的论文"），不需要追问，直接开始：

1. **研究主题**：具体的研究方向/问题/技术名称
2. **时间范围**（可选）：关注哪个时间段的论文？默认为近3-5年，经典开创性工作不受时间限制
3. **侧重点**（可选）：偏理论/偏应用/偏某个子方向？
4. **排除范围**（可选）：有没有明确不想包含的子方向？

---

## 第一步：系统性论文检索

文献综述的质量取决于检索的覆盖面。**必须联网搜索**，不能仅靠已有知识。

### 并行检索策略

使用子Agent并行检索来提高效率。建议的分工：

- **子Agent 1 — arXiv检索**：通过arXiv API系统性检索相关论文，获取标题、作者、摘要、发布时间、引用关系
- **子Agent 2 — 学术搜索与综述发现**：通过Google Scholar、Semantic Scholar等发现高引论文、已有survey论文、关键研究组
- **子Agent 3**（复杂领域才需要）：补充检索，如GitHub上的awesome-list、领域顶会的accepted paper list、研究组主页、技术博客解读

**子Agent联网工具使用指南**（直接写入每个子Agent的prompt中）：

每个子Agent的prompt中必须包含以下联网指引：

> 你需要联网获取论文信息。使用以下工具：
> - **WebSearch**：搜索论文、发现信息来源
> - **WebFetch**：访问具体URL提取内容
> - **arXiv API**：`curl -s "https://export.arxiv.org/api/query?search_query=all:关键词1+AND+all:关键词2&start=0&max_results=30&sortBy=submittedDate&sortOrder=descending"`，返回XML格式，包含标题、作者、摘要、发布日期、PDF链接。可按需调整关键词组合和结果数量
> - **Semantic Scholar API**：`curl -s "https://api.semanticscholar.org/graph/v1/paper/search?query=关键词&limit=20&fields=title,authors,year,abstract,citationCount,venue"` 获取论文及引用数
> - 多次搜索、多个关键词组合，不要只搜一次就放弃
> - 找到关键论文后，用WebFetch读取论文页面获取更多细节
> - 注意发现该领域已有的survey/review论文，它们是重要参考

prompt要描述目标（"获取""调研""了解"），让子Agent自主判断最佳获取方式。

### 检索来源优先级

| 来源 | 用途 |
|------|------|
| arXiv | 最新预印本，覆盖CS/AI/物理/数学等 |
| Semantic Scholar | 引用网络、高引论文发现 |
| Google Scholar | 广泛覆盖，发现跨领域工作 |
| 顶会论文集（NeurIPS, ICML, ICLR, ACL, CVPR等） | 经过同行评审的高质量工作 |
| GitHub awesome-list | 社区整理的论文列表，发现遗漏 |
| 已有survey论文 | 参考其分类体系和覆盖范围 |

### 检索充分性自检

检索完成后检查：
- 该领域的开创性工作（seminal papers）是否找到？
- 高引论文（citation > 100 或该领域的相对高引）是否覆盖？
- 最新进展（近6-12个月）是否包含？
- 主要研究组/团队的代表作是否涵盖？
- 不同技术路线/方法流派是否都有覆盖？

论文不够就再补搜。宁可多收集，后面再筛选。

## 第二步：论文分类与体系构建（Taxonomy）

这是综述的骨架，决定了报告的组织结构。

### 分类策略

根据领域特点选择最合适的分类维度，常见方式：

**按方法/技术路线分**：最常用。比如"基于规则的方法 vs 基于学习的方法 vs 混合方法"。

**按问题/任务分**：适用于一个大方向下有多个子任务的情况。

**按时间演进分**：适用于方法有明显代际演进的领域（如第一代→第二代→第三代）。

**混合分类**：先按大的技术路线分，每个路线内部再按子问题或时间细分。

### 分类要求

- 分类体系要**互斥且穷尽（MECE）**：每篇论文应该能归入某一类，类别之间不应大面积重叠
- 如果某篇论文跨多个类别，归入最核心的那个类别，在其他类别中交叉引用
- 分类层级不超过3级，避免过于碎片化
- 画出分类树（用Markdown缩进或表格呈现），让读者一眼看清全貌

---

## 第三步：各类方法深度解读

这是报告的主体部分。按照分类体系，逐一介绍每类方法。

### 每类方法需要覆盖的内容

**核心思想**：这类方法的基本思路是什么？解决问题的关键insight是什么？

**代表性论文**：列出该类别中最重要的3-5篇论文，每篇包括：
- 论文标题、作者、发表年份与venue
- 核心贡献（用1-3句话概括，不要抄摘要）
- 方法要点（关键技术细节，但不需要完整复现推导）
- 实验结果亮点（在哪些benchmark上取得了什么效果）

**方法演进脉络**：这类方法内部是如何演进的？后续工作相比前驱工作改进了什么？

**优势与局限**：这类方法的优点是什么？固有的局限性是什么？适用场景和不适用场景分别是什么？

### 篇幅

每个主要类别1500-4000字，视论文数量和复杂度调整。整个第三步总计5000-15000字。关键论文值得展开讲，次要论文可以一笔带过。

---

## 第四步：横向对比分析

将不同类别的方法拉到一起进行系统性对比。

### 对比维度

根据领域特点选择对比维度，常见的包括：

| 维度 | 说明 |
|------|------|
| 性能 | 在主流benchmark上的表现（列表格） |
| 计算效率 | 训练/推理的时间和资源开销 |
| 数据需求 | 需要多少数据？什么类型的数据？标注要求？ |
| 可扩展性 | 能否scale到更大的问题规模 |
| 可解释性 | 结果是否可解释、可调试 |
| 泛化能力 | 跨域/跨任务的迁移能力 |
| 实际部署 | 工程落地的难度和成熟度 |

### 对比方式

- **对比表格**：用表格呈现各方法在关键维度上的对比，一目了然
- **定性分析**：表格之外，用文字分析各方法的trade-off，什么场景下该选什么方法
- **Benchmark汇总**：如果该领域有公认的benchmark，汇总各方法的数值对比

### 篇幅

2000-5000字。

---

## 第五步：研究趋势与开放问题

这是综述的升华部分，体现作者的洞察力。

### 趋势分析

- **时间线上的方法演变**：从早期到现在，整个领域的研究范式经历了怎样的变迁？
- **当前热点**：最近1-2年，哪些方向的论文最密集？社区最关注什么问题？
- **跨领域融合**：有没有从其他领域引入的新思路正在产生影响？

### 开放问题与未来方向

- **未解决的核心难题**：这个领域目前最大的瓶颈是什么？
- **有前景的新方向**：基于现有趋势，哪些方向可能产生突破？
- **实际应用的gap**：学术研究和实际应用之间的差距在哪里？
- **需要的基础设施**：是否缺少好的benchmark、数据集、评估标准？

### 篇幅

1500-3000字。

---

## 写作风格

这是一份学术综述，但不应该是枯燥的论文堆砌。目标是让读者在一篇文章里**快速建立对整个领域的全景认知**。

### 核心原则

**叙事驱动，不是罗列驱动**：不要写成"论文A做了X，论文B做了Y，论文C做了Z"的流水账。要讲清楚方法之间的内在联系——为什么会从A发展到B？B解决了A的什么问题？C又带来了什么新的思路？

**精准概括**：每篇论文的核心贡献用1-3句话说清楚。读者看完你的概括，应该能判断是否需要去读原文。

**敢下判断**：鼓励给出对各方法优劣、领域趋势的判断，但每个判断必须有依据。是推测的明确标注。

**面向研究者实用**：读者读完综述后，应该能回答：这个领域现在做到什么程度了？主流方法有哪些？各自优缺点是什么？我如果要做这个方向，应该从哪里切入？

### 避免的问题

- 不要简单罗列论文摘要，要有自己的归纳和理解
- 不要回避方法的局限性，诚实指出每种方法的不足
- 不要遗漏重要的相关工作，覆盖面是综述的基本价值
- 搜不到的信息诚实标注「该信息暂缺」，绝不编造
- 避免AI味套话："综上所述"、"值得注意的是"、"不难发现"、"在当今XX快速发展的时代"

## 第六步：生成PDF报告

报告写完后，使用本Skill自带的 `scripts/md_to_pdf.py` 脚本将Markdown转为排版精美的PDF。

### 转换流程

1. **先完成Markdown稿件**：将完整报告写为标准Markdown格式，保存为 `[研究主题]_paper_survey.md`
2. **安装依赖**（如未安装）：`pip install weasyprint markdown --break-system-packages`
3. **运行转换脚本**：
   ```bash
   python [skill目录]/scripts/md_to_pdf.py input.md output.pdf --title "研究主题" --subtitle "Paper Survey" --author "作者名"
   ```
4. 脚本会自动生成中间HTML文件（便于调试）和最终PDF

### Markdown写作注意事项

为了让脚本正确解析并生成最佳PDF效果：

- 第一行用 `# 标题` 作为报告标题（会自动用于封面）
- 紧接标题后可用 `> 研究领域：... | 时间范围：... | 论文数量：...` 格式写元信息行，会被提取到封面
- 用 `##` 作为主要章节标题
- 用 `###` 和 `####` 作为子章节
- 表格使用标准Markdown表格语法
- 引用使用 `>` 语法
- 论文引用格式：**作者 (年份)**. 标题. *Venue*.

### 报告结构模板

```
封面页

一、引言
[研究背景、调研动机、综述范围说明，500-1000字]

二、分类体系（Taxonomy）
[分类树 + 分类逻辑说明，500-1000字]

三、各类方法详解
  3.1 方法类别A
    3.1.1 核心思想
    3.1.2 代表性论文
    3.1.3 方法演进
    3.1.4 优势与局限
  3.2 方法类别B
  ...
[5000-15000字]

四、横向对比
[对比表格 + 定性分析，2000-5000字]

五、研究趋势与开放问题
[趋势分析 + 未来方向，1500-3000字]

六、结论
[核心发现总结，500-1000字]

参考文献
[完整的论文引用列表]
```

### 文件命名和交付

PDF文件命名为 `[研究主题]_paper_survey.pdf`，保存到用户的工作目录中。
